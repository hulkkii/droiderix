Privacy Policy

Droiderix is local-first. We do not operate a Droiderix account, inference proxy, analytics service, or backend that receives your prompts, files, conversations, credentials, or model output.

Scope

This policy covers the Droiderix Android Play Store V1 release, package com.droiderix.app. Data can leave your device when you choose hosted inference, use a network tool, approve an external Mini App source, install a local model, open an external link, or export or share data.

Data stored on your device

Droiderix stores local workspaces and files, chat history, drafts, run state, agent memories, settings, Mini App permissions, and installed local models in app-managed storage.

Provider authentication tokens and API keys use Android secure storage. Android automatic app backup is disabled. Droiderix creates an export file only when you request one; exported and shared files are then under your control.

Hosted inference providers

When you select ChatGPT or Opper Wallet, the app connects directly to that provider. A request can include your prompt, relevant conversation history, agent instructions, model and runtime settings, tool definitions and results, attachments, and file contents the agent reads. Droiderix does not receive or proxy the request.

Opper Wallet is an inference gateway and may route requests to your selected underlying model provider. Provider retention, training, regional processing, and account practices are controlled by that provider.

Other network connections

  • Context7 tools send library names and documentation queries to context7.com.
  • HTTP fetch and web-to-Markdown tools connect to URLs requested by you or selected by the agent. The destination receives the request URL, headers, body, and normal connection information.
  • Mini Apps can load resources from listed content delivery and font services. Other detected external origins require in-app approval.
  • Installing Gemma-4-E2B downloads the model from Hugging Face. Inference with the installed LiteRT model stays on-device.
  • Authentication pages, policies, the issue tracker, and other external links open outside the app and follow the destination's policies.

Droiderix does not add its own analytics or advertising identifiers to these requests.

V1 features not enabled

Play Store V1 does not enable remote or SSH workspaces, remote terminals, GitHub Copilot, custom API providers, remote agent backends, notification reading, calendar or location tools, Termux, scheduled or background agent automation, or provider-hosted cloud tools such as file search and code interpreter.

Permissions

Play Store V1 requests permissions needed for internet access, vibration, foreground data transfer, and user-requested exports on older Android versions. It does not request notification-listener, location, calendar, camera, microphone, or contacts permissions.

Retention and deletion

Local data remains until you delete it in Droiderix, clear app storage, or uninstall the app. Logging out removes credentials for that provider. Exported backups and files shared to other apps must be deleted separately.

Data sent to an inference provider or website is retained under that service's policies and account controls. Deleting local Droiderix data does not delete provider-side data.

Analytics, advertising, and sale

Droiderix does not include an analytics SDK, advertising SDK, or third-party crash-reporting SDK. We do not sell personal data. Google Play and Android may process installation and diagnostic information under Google's policies independently of Droiderix.

Changes

We may update this policy when the Play Store feature set or data handling changes. The effective date identifies the current version.

Privacy questions: open an issue on the Droiderix GitHub repository. Do not include credentials, private code, prompts, or other sensitive information in issues.
